If I update this source, what HTMLs need review? Reverse dependency map — change-impact view

Read this view as: "If the file on the left changes, the HTMLs on the right need to be opened, reviewed, and likely regenerated." Items marked Primary pull content directly from that source — they'll be wrong if not updated. Items marked Secondary reference it indirectly. Downstream items re-use the affected tier and should at least be spot-checked.

Per Lecture / Learning Objectives / end##_LO_*.docx

Learning-objective doc for a single lecture

Highest-impact source
LO docs are the spine of the entire 3-hub stack for that lecture. Every downstream page is organized around these objectives — an objective being added, renumbered, or reworded cascades everywhere.

Primary (must update)

  • lecture-guides/end##-<slug>.html — sidebar, progress counter, #obj1…#objN sections, #cheatsheet, #quiz all match LO headings
  • active-lesson-drills/end##_active_recall.html — drill questions are seeded per objective
  • active-lesson-drills/end##_key.html — answer key mirrors drill
  • testing-drills/end##-hub1-<slug>.html
  • testing-drills/end##-hub2-<slug>.html
  • testing-drills/end##-hub3-<slug>.html (vignettes anchored to LOs)

Secondary (review)

  • lecture-guides/index.html — lecture-card blurb text
  • active-recall-hub.html — subject-tab blurbs referencing that lecture
  • StudyPlanner.html — task checkboxes per lecture
  • resource-mapping/endocrine_mapping.html — LO column in the matrix
  • resource-mapping/endocrine_resource_hub.html

Downstream / admin

  • hub-update-tracker/status.json — rerun scan.py to refresh completeness
  • endALL-LO-All-Overall Module Objectives.docx — keep master roll-up in sync
Per Lecture / end##_<topic> / *.pdf

Lecture slide deck for a single lecture

Highest-impact source
The slide deck is the authoritative content the Hub 1 guide teaches against. Wording tweaks or slide reorderings don't usually ripple, but new slides, removed topics, or changed mechanisms absolutely do.

Primary (must update)

  • lecture-guides/end##-<slug>.html — all prose, diagrams, #cheatsheet, #references cite specific slides
  • lecture-guides/end##-*.html #audit section — slide-by-slide source audit table
  • active-lesson-drills/end##_active_recall.html — "Lecture ##" references
  • active-lesson-drills/end##_key.html

Secondary (review)

  • testing-drills/end##-hub1|hub2|hub3-*.html — question stems may cite specific slide facts
  • resource-mapping/endocrine_mapping.html — slide-level references

Downstream / admin

  • Per Lecture/Learning Objectives/end##_LO_*.docx — check if LOs need to change too
  • hub-update-tracker/content-cache.json — invalidate cache
Per Subject / *.docx (Histo / Path / Physio / Pharm / Biochem)

Subject-level review guide

Cross-cutting source
Subject review guides cut across lectures — a Path update can touch every lecture that has a pathology component (end08, end11, end16, end20, end24).

Primary (must update)

  • Every lecture-guides/end##-*.html whose subject tag matches — e.g. a Pharmacology update hits end13, end17, end18, end21, end22, end25
  • active-recall-hub.html — the matching subject tab (#physio, #path, #histo, #biochem, #drt)

Secondary (review)

  • testing-drills/*-hub2|hub3-*.html for affected lectures

  • StudyPlanner.html — if checklist headings changed

Recommended change workflow

  1. Identify the source tier (LO, slide, subject, workbook, Anki, mapping, practice, schedule) — use the cards above.
  2. Update the Primary column first — these pages pull content directly and will be visibly wrong.
  3. Spot-check Secondary — usually small copy tweaks (blurbs, tab labels, matrix cells).
  4. Re-run the trackers in hub-update-tracker/ (python scan.py, python mapping_tracker.py) to refresh JSON feeds the dashboards read.
  5. Regenerate via the skills — hub-builder / hub-index-generator / hub3-generator / study-planner-generator if the change is structural rather than content-level.
